Overview

Implemented an end-to-end FPGA image display system on the Digilent Basys 3 (Xilinx Artix-7). The design reads raw 320×240 frames from a microSD card via SPI and renders them on a VGA monitor in real time. All RTL is written in synthesizable Verilog and built with Xilinx Vivado.

Demo

Final project for 2110363 Hardware Synthesis Lab I, Chulalongkorn University. Video frames are 320×240 pixels at 12-bit colour stored as raw frames on the SD card and read over SPI (50 MHz). Output is driven over VGA; recorded playback was slightly sped up to match the original music video.

Watch on YouTube: https://www.youtube.com/watch?v=p0MyfT0LiCs

Key Features

  • MicroSD Interface — SPI-based SD master controller (sd_master.v) for block-level reads.
  • VGA Signal Generation — Custom VGA sync module producing H-sync, V-sync and blanking intervals.
  • Image Rendering — 320×240 pixel pipeline from SD block reads to pixel output.
  • Image Switching — Hardware button to switch between stored images on the SD card.
  • Top-Level Integration — Modular top module (TopModule.v) that ties Feeder, Controller, VGA, and SD subsystems together.

Technical Stack

  • Verilog — RTL design, simulation, and synthesis (majority of code).
  • Python / Jupyter — Image preprocessing and binary/COE generation for SD content.
  • Tcl — Vivado project scripting and build automation.
  • Tools: Xilinx Vivado, Basys 3 board, XDC constraint for pin mapping.

Repository Structure

├── Controller.v      # System state machine & control logic

sim/                  # Testbenches and simulation files

top.xdc               # Basys 3 pin constraint file

What I Learned

  • Implemented SPI in Verilog to interface with a microSD card without IP cores.
  • Designed a VGA controller from first principles, handling pixel clock and blanking correctly.
  • Built a robust data pipeline from SD block reads → pixel buffering → VGA raster output.
  • Used Python tooling to prepare image data and automated Vivado flows with Tcl.
